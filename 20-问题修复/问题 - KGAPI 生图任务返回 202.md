# 问题 - KGAPI 生图任务返回 202

## 现象

调用 `https://api.kgapi.net/v1/images/generations` 后，没有直接返回图片，而是返回 `202`。

## 原因

这个网关的生图接口采用异步任务制。创建任务后会先返回 `queued` 状态与 `poll_url`，后续需要继续轮询任务结果。

## 解决办法

1. 先发起生图请求
2. 读取返回中的 `poll_url`
3. 定时轮询任务状态
4. 等待 `status = succeeded`
5. 从返回的 `data[0]` 中取 `b64_json` 或图片 URL

## 已验证结论

- `/v1/models` 可正常访问
- 公开返回的标准生图模型名为 `gpt-image-2`
- 生图成功后可通过任务结果中的图片地址下载成 PNG

## 以后如何避免

- 不要把 `202` 误判为失败
- 不要把模型名写成 `image2`，优先使用标准 ID `gpt-image-2`
- 新接入其他图像网关时，先确认是同步回图还是异步任务制
