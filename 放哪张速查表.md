# 放哪张速查表

不知道放哪时，先记住一句：

**不确定时，先放 `00-收件箱`。**

## 快速判断

### 放到 `00-收件箱`

适合这些内容：

- 刚聊完，觉得有价值，但还没想好怎么分类
- 临时记录、碎片想法、候选内容
- 自动脚本刚提取出来的内容
- 你不想现在花时间判断归档位置

一句话判断：

**先留住，再整理。**

### 放到 `10-配置中心`

适合这些内容：

- API 配置
- 模型默认值
- 全局规则
- 插件设置
- 软件路径
- 环境变量

一句话判断：

**以后还要照着这个设置再配一次。**

### 放到 `20-问题修复`

适合这些内容：

- 报错
- 失败原因
- 排查过程
- 修复办法
- 恢复流程

一句话判断：

**这是一个以后可能还会再遇到的坑。**

### 放到 `30-工作流`

适合这些内容：

- 一套可重复执行的步骤
- 日常协作方法
- 工具联动流程
- 自动化流程
- 资料整理方法

一句话判断：

**这是以后还能反复照着做的方法。**

### 放到 `40-专题研究`

适合这些内容：

- 长期会持续补充的主题
- 某个方向的系统化理解
- 一组彼此相关的页面
- 需要不断积累的研究或实践主题

一句话判断：

**这不是单条记录，而是一整个会继续长大的主题。**

## 最实用的偷懒用法

如果你不想自己判断，直接对 Codex 说：

- `把这段记下来`
- `帮我归档这段`
- `把这段放到合适的位置`
- `把这个写成正式笔记`

然后由 Codex 来决定该进：

- `00-收件箱`
- `10-配置中心`
- `20-问题修复`
- `30-工作流`
- `40-专题研究`

## 最后再记一句

**收件箱不是混乱，而是缓冲区。**
